# Goblin Studios — New Site

Official website for Goblin Studios, a premium game development studio specialized in Unreal Engine. Built as a full rebuild from the original Webflow site, now running on a modern, self-hosted React stack.

---

## Tech Stack

| Tool | Purpose |
|---|---|
| [React 19](https://react.dev/) | UI framework |
| [TypeScript](https://www.typescriptlang.org/) | Type safety |
| [Vite 7](https://vitejs.dev/) | Build tool and dev server |
| [TanStack Router](https://tanstack.com/router) | File-based routing |
| [Tailwind CSS v4](https://tailwindcss.com/) | Styling |
| [shadcn/ui](https://ui.shadcn.com/) | Base UI components |
| [Bun](https://bun.sh/) | Package manager and runtime |

---

## Project Structure

```
src/
├── assets/          # Images and static media (team photos, key art)
├── components/
│   ├── site/        # Project-specific components (Header, Footer, Layout...)
│   └── ui/          # Generic shadcn/ui components
├── hooks/           # Custom React hooks
├── lib/             # Utilities and helpers
├── routes/          # One file per page (TanStack Router convention)
│   ├── __root.tsx   # Root layout (navbar, footer, global providers)
│   ├── index.tsx    # Home page
│   ├── contact.tsx  # Contact page (with form)
│   ├── services.tsx # Services page
│   ├── everlen.tsx  # Everlen IP page
│   └── chamado.tsx  # O Chamado do Herói IP page
├── main.tsx         # App entry point
├── router.tsx       # Router configuration
└── styles.css       # Global styles and design tokens
public/
└── videos/          # Looping background videos (hero, logo marquees)
```

## Getting Started

### Requirements

- [Bun](https://bun.sh/) v1.0 or higher
- Node.js is **not** required — Bun handles everything

### Install dependencies

```bash
bun install
```

### Run dev server

```bash
bun run dev
```

Opens at `http://localhost:8080`

### Build for production

```bash
bun run build
```

Output goes to `dist/`.

### Preview production build

```bash
bun run preview
```

## Editing Guide

### Changing page content
Each page is a self-contained file inside `src/routes/`. Open the relevant file and edit the text, arrays, or components directly — no CMS needed.

### Adding/removing nav links
Edit the `NAV` array in `src/components/site/Header.tsx`.

### Updating team members
Edit the `FOUNDERS` array in `src/routes/index.tsx`. Each entry has `name`, `role`, and `img` fields.

### Updating client/logo marquee
The marquee videos are in `public/videos/`. Replace `logos-left.mp4` and `logos-right.mp4` to update the scrolling logos.

### Updating the hero video
Replace `public/videos/hero.mp4` with a new video file. Keep the same filename or update the `src` in the `Hero` component inside `src/routes/index.tsx`.

### Global styles and design tokens
All CSS variables (colors, fonts, spacing) are defined in `src/styles.css`.
